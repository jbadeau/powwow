OpenAjax Hub 2.0 Overview

Relationship of OpenAjax Hub 1.0 and OpenAjax Hub 2.0

OpenAjax Hub 2.0 represents an update to OpenAjax Hub 1.0 in order to address the requirements of secure mashups:

  • OpenAjax Hub 1.0 defines a industry standard API for pub/sub messaging (i.e., publish/subscribe) between JavaScript components that are used within the same browser frame.
  • OpenAjax Hub 2.0 extends Hub 1.0 to add secure mashups where HTML+JavaScript components (i.e., widgets) can be isolated into secure sandboxes (using IFRAMEs typically) and can communication each other via pub/sub messaging using OpenAjax Hub 2.0, where Hub 2.0 allows messages to be passed securely across browser frames.

Working Together: OpenAjax Hub 2.0 and OpenAjax Metadata

OpenAjax Hub 2.0 and OpenAjax Metadata were designed in coordination to ensure that the Web Widgets defined using OpenAjax Metadata would have the required information necessary to allow those widgets to be used conveniently within a secure mashup framework that is built on top OpenAjax Hub 2.0.

  • OpenAjax Hub 2.0 - OpenAjax Hub 2.0 specifies industry standard APIs for a secure mashup runtime that runs in today's browsers (and that has a forward-looking architecture for future browsers). In addition to the Hub 2.0 specification which defines standard APIs, OpenAjax Alliance is also producing a commercial-grade open source reference implementation of Hub 2.0.
  • OpenAjax Metadata - In coordination with Hub 2.0, OpenAjax Alliance is defining OpenAjax Metadata for Widgets.

These two initiatives help the industry realize the potential of mashup technologies to enable user-generated situational applications, where applications are assembled in the browser by end users by dragging various widgets onto a canvas and connecting widgets together. These initiatives address the two main inhibitors to widespread deployment of mashups:

  • Mashup security - The main goal with user mashups is to allow users to discover and use 3rd party widgets within a composite applications. Most 3rd party widgets are trustworthy, but malicious widgets will exist and there are significant security risks with mashups since widgets are assembled together within the same Web page. A poorly designed mashup environment might allow malicious widgets to gain access to user confidential or company confidential information, or leverage the user's credentials to extract and modify information in company databases. OpenAjax Hub 2.0 addresses these security issues by isolating widgets into "sandboxes" (usually using IFRAMEs) where widgets can only communicate with each other over a managed message bus. Widgets can only receive and send messages that are allowed by the security manager component.
  • Widget interoperability - Today there are dozens of formats for Web Widgets, including OpenSocial Gadgets, Apple Dashboard Widgets, Vista Gadgets, and Yahoo Confabulator widgets. OpenAjax Metadata for Widgets defines an industry standard for Web Widgets for use in mashups.

A widget's view of OpenAjax Hub 2.0

A Web Widget provides either a snippet of HTML that can be placed into an HTML DIV or a URL for a web service that delivers a complete HTML page. In either case, the mashup framework manages the loading and initialization of the widget, including the bootstrapping of the widget's ability to publish and subscribe through Hub 2.0. The widget receives a hubConnectionHandle, which is an opaque JavaScript object through which the widget can invoke:

  • hubConnectionHandle.publish(...) - to broadcast a message that other widgets can receive
  • hubConnectionHandle.subscribe(...) - to subscribe to messages that other widgets might send

For further information, see:

  • OpenAjax Hub 2.0 Specification. In particular, read through the 'Managed Hub Overview' and 'Managed Hub APIs' chapters.
  • OpenAjax Metadata for Widgets. In particular, read through the 'Widget Metadata' chapter.

A mashup framework's view of OpenAjax Hub 2.0

The Sample Mashup Application provides an open source reference implementation for how a mashup application can use OpenAjax Hub 2.0 and OpenAjax Metadata to deliver a secure product for end-user-created mashups.

In the open source reference implementation, the mashup application loads the JavaScript source code for OpenAjax Hub 2.0 and then creates a "Managed Hub Instance", attaching to that instance a Security Manager. The reference implementation uses Hub 2.0 facilities for loading widgets into secure sandboxes (usually using IFRAMEs) and uses Hub 2.0 for its low-level publish/subscribe messaging system to allow widgets to communicate with each other.

  • OpenAjax Hub 2.0 Specification. In particular, read through the 'Managed Hub Overview' and 'Managed Hub APIs' chapters.
